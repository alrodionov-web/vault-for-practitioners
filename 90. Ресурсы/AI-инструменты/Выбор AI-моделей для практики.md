# Исследование AI-моделей для практики

## Цель проекта

Сравнить AI-модели по соотношению цена/производительность для трёх сценариев работы помогающего практика:

1. **agent** - установка ПО в среду и отладка
2. **think** - размышления о позиционировании или здоровье
3. **text** - написание постов для соцсетей и другого контента

## Сценарии работы

### 1. Agent (установка ПО и отладка)
**Требования:**
- Техническая экспертиза
- Понимание систем, команд, зависимостей
- Отладка, анализ ошибок
- Структурированное решение проблем

**Критерии оценки:**
- Точность технических решений
- Способность читать и интерпретировать документацию
- Эффективность в решении проблем
- Стоимость выполнения задач

### 2. Think (размышления)
**Требования:**
- Глубокий анализ, рефлексия
- Понимание психологических концепций
- Системное мышление
- Творческий подход

**Критерии оценки:**
- Глубина проработки тем
- Качество структурирования мыслей
- Практическая применимость идей
- Стоимость сессии размышлений

### 3. Text (написание контента)
**Требования:**
- Качественный текст на русском
- Маркетинговый подход
- Эмпатия к аудитории
- Соответствие бренду

**Критерии оценки:**
- Качество текста (грамотность, стиль)
- Соответствие целевой аудитории
- Эффективность для вовлечения
- Стоимость создания контента

# Тестирование моделей для сценария Agent

## Тестовые задачи

### Задача 1: Установка и настройка ПО
**Описание:** Установить утилиту командной строки `gws` для работы с Google Calendar API

**Требования:**
- Понимание системных зависимостей
- Работа с менеджерами пакетов (brew, pip)
- Настройка OAuth2 авторизации
- Отладка ошибок доступа

### Задача 2: Отладка окружения
**Описание:** Решить проблему с переменными окружения для Python проекта

**Требования:**
- Диагностика проблем с путями
- Работа с виртуальными окружениями
- Исправление конфликтов зависимостей

## Критерии оценки

### Технические (0-10 баллов)
1. **Точность команд** - правильность предложенных команд
2. **Понимание контекста** - учет ОС, версий, зависимостей
3. **Диагностика** - способность анализировать ошибки
4. **Безопасность** - избегание опасных команд

### Практические (0-10 баллов)
1. **Структура ответа** - четкость, последовательность
2. **Объяснения** - понятные пояснения для пользователя
3. **Альтернативы** - предложение разных подходов
4. **Проактивность** - предупреждение о потенциальных проблемах
