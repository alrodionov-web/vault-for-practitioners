# Скилл: Группы

Создание и ведение групповых программ: тренинги, марафоны, мастер-классы, групповая терапия.

## Роутинг по намерению

- "создай программу", "новый тренинг", "марафон", "групповая программа" -> Создать программу
- "спланируй занятие", "план встречи", "следующее занятие" -> План занятия
- "участники", "кто в группе", "посещаемость" -> Управление участниками
- "раздатка", "материалы", "домашнее задание для группы" -> Материалы

## Структура данных

```
21. Группы/
  [Название программы]/
    программа.md        <- общая структура, модули, цели
    участники.md        <- список, посещаемость, заметки
    занятия/
      01 YYYY-MM-DD.md  <- план и заметки по занятию
      02 YYYY-MM-DD.md
    материалы/          <- раздатки, упражнения
```

## Подкоманды

### Создать программу

1. Спросить:
   - Название и тема
   - Для кого (целевая аудитория)
   - Формат: тренинг / марафон / мастер-класс / терапевтическая группа
   - Количество занятий и длительность каждого
   - Онлайн или офлайн
2. Предложить структуру модулей:
   - Цель каждого модуля
   - Ключевые темы
   - Логика последовательности (от простого к сложному / по этапам)
3. Показать и спросить: "Так? Поправить?"
4. Создать папку и файлы:
   - `21. Группы/[Название]/программа.md`
   - `21. Группы/[Название]/участники.md` (пустой шаблон)
   - `21. Группы/[Название]/занятия/` (пустая папка)

### План занятия

1. Прочитать `программа.md` - определить какое занятие по счету, какая тема
2. Предложить структуру:

```
# Занятие N: [тема]
Дата: YYYY-MM-DD
Длительность: N мин

## Разминка (N мин)
- [упражнение / вопрос для обсуждения]

## Основная часть (N мин)
### Блок 1: [тема]
- [формат: мини-лекция / упражнение / работа в парах / обсуждение]
- [материалы]

### Блок 2: [тема]
- [формат]

## Обсуждение и рефлексия (N мин)
- [вопросы для обсуждения]

## Завершение (N мин)
- [итоги, домашнее задание]

## Заметки после занятия
- Динамика группы:
- Что сработало:
- Что изменить:
```

3. Показать и спросить подтверждение
4. Сохранить в `занятия/NN YYYY-MM-DD.md`

### Управление участниками

1. Прочитать `участники.md`
2. Подкоманды:
   - "добавь участника" -> добавить в список
   - "отметь посещаемость" -> обновить таблицу
   - "как группа" -> показать сводку (кто ходит, кто пропускает)

Формат `участники.md`:

```
# Участники: [Название]

| Имя | Контакт | Занятие 1 | Занятие 2 | ... | Заметки |
|-----|---------|-----------|-----------|-----|---------|
| [имя] | [контакт] | + | - | | [заметки] |
```

### Материалы

1. Спросить: для какого занятия, какой формат (упражнение / чеклист / теория)
2. Прочитать план занятия
3. Создать материал и сохранить в `материалы/`

## Правила

- Имена участников - инициалы или имена без фамилий (если не публичная группа)
- Не давать клинических рекомендаций для участников
- Хронометраж должен сходиться (сумма блоков = общая длительность)
