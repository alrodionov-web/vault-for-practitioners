# Скилл: Ежедневное планирование

## CRITICAL: Правила (прочитай перед выполнением)

- Путь к дневной заметке: `01. Планирование/YYYY/MM/YYYY-MM-DD.md`
  - НЕПРАВИЛЬНО: `01. Планирование/2026-04-14.md`
  - НЕПРАВИЛЬНО: `01. Планирование/2026/2026-04-14.md`
  - ПРАВИЛЬНО: `01. Планирование/2026/04/2026-04-14.md`
- ОБЯЗАТЕЛЬНО спросить пользователя о корректировке перед сохранением
- НЕ сохранять файл без явного "да" от пользователя
- Ответ максимально короткий: факты + вопрос
- НЕ ДОДУМЫВАТЬ содержание встреч и задач. Если в календаре написано "Встреча с кураторами" - писать только "Встреча с кураторами", БЕЗ подзадач типа "подготовить материалы", "составить повестку". Показывать ТОЛЬКО то, что есть в источнике.

---

## Утро: Статус на день

### Шаг 1. Собрать контекст

1. Прочитать последний недельный план из `01. Планирование/` (файл `Неделя NN.md`)
2. Найти задачи на сегодня
3. Получить расписание (по приоритету):
   a. ICS-ссылка из `ZZ. Служебное/calendar-config.md` (если настроена) -> загрузить события на сегодня
   b. Скриншот от пользователя: "Сделайте скриншот расписания на сегодня (Mac: Cmd+Shift+4, Windows: Win+Shift+S) и вставьте сюда."
   c. Текстом: "Какие встречи и сессии сегодня?"

Если недельного плана нет -> просто спросить: "Какие задачи и встречи сегодня?"

### Шаг 2. Проверить паузу

Проверить логи `ZZ. Служебное/sessions/` - когда было последнее общение.

Если пауза 2+ дня:

> "Мы не общались с [дата]. На [пропущенные дни] было запланировано:
> - [задача 1]
> - [задача 2]
>
> Что удалось сделать? Не страшно если нет - просто обновим план."

Отметить выполненное/невыполненное в недельном плане.

### Шаг 3. Показать день

**CRITICAL:** Проверить текущее время. Показывать ТОЛЬКО оставшуюся часть дня. Не планировать на прошедшие часы.
- НЕПРАВИЛЬНО: сейчас 13:00, а в плане задачи на 09:00-12:00
- ПРАВИЛЬНО: сейчас 13:00, план начинается с 13:00

Формат ответа:

```
# Сегодня, [день недели] DD.MM (сейчас HH:MM)

## Оставшиеся встречи
- HH:MM - Встреча [из календаря]

## Задачи из недельного плана
- [ ] Задача 1 (~Xч)
- [ ] Задача 2 (~Xч)

## Свободное время (с текущего момента)
- HH:MM-HH:MM (Nч) - окно для работы
```

### CRITICAL: СТОП. Спросить пользователя:

> "Нужно что-то поправить? Добавить, убрать, перенести?"

НЕ переходить к сохранению без ответа. Ждать.

### Шаг 4. Корректировка

Если пользователь хочет поправки:
1. Скорректировать план
2. Показать обновленный вариант
3. Спросить: "Так? Сохраняем?"

### Шаг 5. Сохранение

Только после подтверждения пользователя:
1. Создать папку `01. Планирование/YYYY/MM/` (если не существует)
2. Сохранить как `01. Планирование/YYYY/MM/YYYY-MM-DD.md`
   - НЕПРАВИЛЬНО: `01. Планирование/YYYY-MM-DD.md`
   - ПРАВИЛЬНО: `01. Планирование/YYYY/MM/YYYY-MM-DD.md`

Если пользователь НЕ просил сохранять -> не сохранять, просто показать статус.

## Вечер: Рефлексия (опционально)

Если пользователь спрашивает "как день" или "итоги":

1. Прочитать утренний план (если был) или задачи из недельного плана на сегодня
2. Спросить: "Как прошел день? Что сделано, что нет?"
3. Дополнить заметку дня или обновить недельный план:

```
## Итоги дня
- Сделано: [список]
- Не сделано: [список]
- Заметки: [мысли, инсайты]
```
