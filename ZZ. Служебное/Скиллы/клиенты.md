# Скилл: Клиенты

Управление клиентской базой: профили, записи сессий, динамика, подготовка к супервизии.

## Роутинг по намерению

Определи подкоманду из запроса пользователя:

- "новый клиент", "завести клиента" -> Новый клиент
- "запиши сессию", "заметки по сессии", "провел сессию" -> Запись сессии
- "как дела у [имя]", "динамика", "прогресс клиента" -> Динамика
- "подготовь к супервизии", "кейс для супервизии" -> Подготовка к супервизии
- "сколько клиентов", "нагрузка", "кейслоад" -> Обзор нагрузки
- "подготовься к сессии", "что у нас с [имя]" -> Подготовка к сессии

## Подкоманды

### Новый клиент

1. Спросить псевдоним (НИКОГДА не использовать реальное имя)
2. Создать папку `20. Клиенты/[Псевдоним]/`
3. Создать `Профиль.md` по шаблону:

```
---
description: Профиль клиента [Псевдоним]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# [Псевдоним]

## Основное
- Возраст: [PLACEHOLDER]
- Запрос: [PLACEHOLDER]
- Начало работы: YYYY-MM-DD
- Формат: индивидуальный / онлайн / офлайн

## Цели работы
- [PLACEHOLDER]

## Заметки
```

### Запись сессии

1. Спросить: с кем была сессия (псевдоним)
2. Прочитать `20. Клиенты/[Псевдоним]/Профиль.md`
3. Спросить: "Расскажите коротко, что было на сессии. Можно тезисами, можно голосовым (вставьте текст)."
4. Оформить в структурированную заметку:

```
# Сессия YYYY-MM-DD

## Состояние на входе
- [из рассказа]

## Что делали
- [техники, темы, упражнения]

## Ключевые моменты
- [инсайты, реакции, важное]

## Домашнее задание
- [если было]

## Заметки для себя
- [что обратить внимание, гипотезы]

## На следующую сессию
- [что продолжить, проверить]
```

5. Сохранить как `20. Клиенты/[Псевдоним]/YYYY-MM-DD Сессия.md`
6. Обновить `updated` в `Профиль.md`

### Динамика клиента

1. Прочитать все файлы сессий клиента
2. Показать:
   - Количество сессий, период работы
   - Изменение состояния (первая сессия vs последняя)
   - Повторяющиеся темы
   - Прогресс по целям из Профиля
   - Что менялось в домашних заданиях
3. Спросить: "Хотите обновить цели работы в профиле?"

### Подготовка к сессии

1. Прочитать `Профиль.md` и последние 2-3 сессии
2. Показать краткую сводку:
   - Текущие цели
   - Что было на прошлой сессии
   - Невыполненные домашние задания
   - На что обратить внимание
3. Спросить: "Что-то хотите учесть на сегодняшней сессии?"

### Подготовка к супервизии

1. Спросить: какого клиента обсуждаем и какой вопрос к супервизору
2. Прочитать все сессии клиента
3. Создать АНОНИМИЗИРОВАННУЮ выжимку:
   - Убрать все идентифицирующие детали
   - Заменить имена, места, даты на нейтральные
   - Оставить клинически значимое
4. Структура:
   - Запрос клиента
   - Краткая история работы
   - Что пробовали
   - Где застряли / вопрос к супервизору
5. Сохранить в `20. Клиенты/[Псевдоним]/супервизия YYYY-MM-DD.md`

### Обзор нагрузки

1. Прочитать все папки в `20. Клиенты/`
2. По каждому клиенту: когда последняя сессия, сколько всего сессий
3. Показать:
   - Активные клиенты (сессия за последние 2 недели): N
   - На паузе (нет сессий 2+ недели): N
   - Всего сессий за последний месяц: N
   - Клиенты с долгим перерывом (4+ недели без сессий)

## Правила

- НИКОГДА не использовать реальные имена - только псевдонимы
- Не давать клинических рекомендаций или диагнозов
- Если пользователь вводит реальные данные клиента - предупредить
- `[PLACEHOLDER]` для неизвестного, не выдумывать
