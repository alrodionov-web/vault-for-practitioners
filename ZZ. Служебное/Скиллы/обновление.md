# Скилл: обновление хранилища

## CRITICAL: Правила

- Прочитать `ZZ. Служебное/version.md` ПЕРВЫМ ДЕЛОМ - извлечь текущую версию и URL репозитория
- НЕ трогать файлы, которых НЕТ в `ZZ. Служебное/system-files.txt`
- ВСЕГДА спрашивать подтверждение перед обновлением
- При ошибках загрузки - сообщить пользователю, НЕ пытаться повторять бесконечно

---

## Алгоритм

### Шаг 1. Проверить текущую версию

Прочитать `ZZ. Служебное/version.md`. Найти строку `Текущая: vX.Y`.
Найти строку `Репозиторий: URL` - это базовый URL.

### Шаг 2. Проверить новую версию

Загрузить version.md из репозитория через terminal:

```
curl -s "https://raw.githubusercontent.com/alrodionov-web/vault-for-practitioners/main/ZZ.%20%D0%A1%D0%BB%D1%83%D0%B6%D0%B5%D0%B1%D0%BD%D0%BE%D0%B5/version.md"
```

Из ответа найти строку `Текущая: vX.Y` - это последняя доступная версия.

Если совпадает с локальной -> сказать "У вас последняя версия vX.Y" -> СТОП.

### Шаг 3. Показать изменения

Из скачанного version.md показать пользователю секцию "Что нового" для версий новее текущей.

**CRITICAL: СТОП. Спросить:** "Обновить до vX.Y? Будут заменены только системные файлы (скиллы, правила, шаблоны). Ваши заметки, клиенты, планы и цели НЕ будут затронуты."

Без подтверждения - не продолжать.

### Шаг 4. Скачать список файлов

```
curl -s https://raw.githubusercontent.com/alrodionov-web/vault-for-practitioners/main/ZZ.%20%D0%A1%D0%BB%D1%83%D0%B6%D0%B5%D0%B1%D0%BD%D0%BE%D0%B5/system-files.txt
```

### Шаг 5. Обновить файлы

CRITICAL: Использовать ТОЛЬКО terminal (curl -o) для записи. НЕ использовать edit/write инструменты - это вызывает построчное сравнение и занимает вечность.

Для каждого файла из списка:
1. URL-кодировать путь (пробелы -> `%20`, кириллица -> `%XX`)
2. Создать папку если нужно и скачать одной командой:

```
mkdir -p "$(dirname "ПУТЬ")" && curl -s -o "ПУТЬ" "https://raw.githubusercontent.com/alrodionov-web/vault-for-practitioners/main/URL_КОДИРОВАННЫЙ_ПУТЬ"
```

Можно объединять несколько файлов в одну команду через `&&`.

### Шаг 6. Подтвердить

Сказать пользователю: "Хранилище обновлено до vX.Y. Обновлено N файлов."

---

## Формат ответа при проверке

```
# Проверка обновлений

- Текущая версия: v0.1
- Доступная версия: v0.2

## Что нового в v0.2
- ...

Обновить? Ваши данные не будут затронуты.
```
