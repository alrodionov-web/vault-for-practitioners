# Скилл: Недельное планирование

## CRITICAL: Правила (прочитай перед выполнением)

- Путь к недельному плану: `01. Планирование/YYYY/MM/Неделя NN.md` (NN - номер недели ISO)
  - НЕПРАВИЛЬНО: `01. Планирование/Неделя 16.md`
  - ПРАВИЛЬНО: `01. Планирование/2026/04/Неделя 16.md`
- ВСЕ предложения показывать пользователю и ждать подтверждения
- НЕ сохранять файл без явного "да" от пользователя
- Не планировать больше 70% свободного времени (буфер)

---

## Процесс

### Фаза 1. Сбор картины (делает агент)

1. Прочитать `ZZ. Служебное/текущий-фокус.md`
2. Прочитать последний недельный план из `01. Планирование/` (для ретро)
3. Прочитать `01. Планирование/Цели.md` (если заполнен) - фокусы и метрики
4. Прочитать логи сессий из `ZZ. Служебное/sessions/` за последние 7 дней
5. Получить расписание из `ZZ. Служебное/calendar-config.md`:
   a. CalDAV -> запрос REPORT на всю неделю (см. `ZZ. Служебное/Скиллы/calendar.md`)
   b. ICS -> загрузить через fetch и распарсить события на неделю
   c. Не подключен -> спросить: "Какие встречи и сессии на этой неделе?" и предложить подключить календарь

### Триггер месячного обзора

Посчитать количество недельных планов с последнего обзора месяца. Если это 5-я, 9-я или 13-я неделя:

> "Прошел месяц с последнего обзора. Хотите посмотреть, как двигаются цели? Или если целей еще нет - поставить их на основе того, чем занимались?"

Если да -> перейти к `ZZ. Служебное/Скиллы/обзор-месяца.md`. Если нет -> продолжить.

### Фаза 2. Ретро прошлой недели

Если есть прошлый недельный план - показать:

```
## Ретро прошлой недели
- Запланировано: N задач, выполнено: M
- Выполнено: [список]
- Не выполнено: [список]

## Над чем работали с агентом
- [из логов сессий: темы, файлы, результаты]
```

### CRITICAL: СТОП. Спросить пользователя:

> "Что из невыполненного переносим на эту неделю? Что уже неактуально?"

Ждать ответа. НЕ переходить к задачам без ответа.

Если первая неделя -> пропустить ретро.

Если % выполнения ниже 50% три недели подряд -> мягко предложить: "Последние недели выполняется меньше половины. Может, стоит планировать меньше задач, но выполнять все?"

### Фаза 3. Задачи на неделю

По каждому проекту/контексту из `Цели.md` (или из `22. Проекты/` и основных папок если целей нет):

> "[Проект/контекст]: какие задачи на эту неделю?"

Собрать все. Спросить:

> "Есть ли что-то вне проектов, что нужно сделать на этой неделе?"

### Фаза 4. Приоритизация

Предложить разделение на must (обязательно) и nice (по возможности).

### CRITICAL: СТОП. Показать предложение и спросить:

> "Так? Хотите поправить?"

Ждать ответа.

Если пользователь затрудняется ("все важно"):

> "Если бы на этой неделе можно было сделать только одну вещь - какую?"

Затем: "А если две?" - и так далее.

Попросить оценить время (0.5ч / 1ч / 2ч / 4ч). Посчитать сумму vs свободные часы:

> "Задач на ~Nч, свободных ~Mч. [Все влезает / Не влезает - что убрать?]"

Must-задачи должны помещаться в 60% свободного времени.

### Фаза 5. Раскидка по дням

На основе расписания:
1. Показать свободные окна по дням
2. Предложить распределение задач по дням
3. Дни с большим количеством встреч -> не нагружать задачами

### CRITICAL: СТОП. Показать и спросить:

> "Вот распределение. Поправить?"

Ждать ответа.

Если нет интеграции с календарем -> не ставить временные блоки, просто записать задачи.

### Фаза 6. Сохранение

Только после подтверждения пользователя:
1. Создать папку `01. Планирование/YYYY/MM/` (если не существует)
2. Сохранить как `01. Планирование/YYYY/MM/Неделя NN.md`
   - НЕПРАВИЛЬНО: `01. Планирование/Неделя 16.md`
   - ПРАВИЛЬНО: `01. Планирование/2026/04/Неделя 16.md`
3. Обновить `ZZ. Служебное/текущий-фокус.md`

---

## Формат выходной заметки

```
# Неделя NN (DD.MM - DD.MM)

## Ретро прошлой недели
- Запланировано: N, выполнено: M (X%)
- Не выполнено: [список]

## Задачи недели

### Must
1. Задача - [проект] (~Xч)
2. Задача - [проект] (~Xч)

### Nice
3. Задача - [проект] (~Xч)

## План по дням

### Понедельник DD.MM
- Встречи: [из расписания]
- [ ] Задача (~Xч)

### Вторник DD.MM
- Встречи: [из расписания]
- [ ] Задача (~Xч)

...

## Итого
- Задач: N (must: M, nice: K)
- Запланировано часов: ~X из ~Y свободных
```
